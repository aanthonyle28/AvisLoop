<task type="auto">
  <name>Task 1: Data layer — getWebDesignClients and getClientMrrSummary</name>
  <files>lib/data/clients.ts</files>
  <action>
Create `lib/data/clients.ts` as a server-only module (add `import 'server-only'` at top).

Define and export the `WebDesignClient` TypeScript type. It must include all existing `Business` fields PLUS:
- `web_project: { id: string; project_name: string; page_count: number | null; status: string; launched_at: string | null; portal_token: string | null } | null`
- `revisions_used_this_month: number` (count of project_tickets created in the current calendar month for this business_id)

**`getWebDesignClients()`:**
- Authenticate with `createClient()` from `@/lib/supabase/server`
- Return `[]` early if not authenticated (same pattern as `getUserBusinessesWithMetadata`)
- Query `businesses` with `.select('*, web_projects(*)')` filtered by `.in('client_type', ['web_design', 'both'])`
- Order by `created_at ascending`
- For each business, count project_tickets where `business_id = business.id` AND `created_at >= start of current UTC month` — do this as a second query with `.gte('created_at', startOfMonth.toISOString())` filtering on business IDs from the first result. Build a map of `businessId -> count`, then attach `revisions_used_this_month` to each business row.
- `web_projects(*)` returns an array; take index `[0]` or null as `web_project`.
- Return typed `WebDesignClient[]`

**`getClientMrrSummary()`:**
- Authenticate; return `0` if not authenticated
- Query businesses with `.select('monthly_fee').in('client_type', ['web_design', 'both']).eq('status', 'active')`
- Sum `monthly_fee` values (skip nulls), return as `number`
- This is a separate exported async function, not bundled into getWebDesignClients, so the page can fetch both in parallel.

**`lib/validations/client.ts`:**
Create Zod schema `clientUpdateSchema` covering the fields the operator can edit from the detail view (CRM-04):
- `owner_name`: z.string().max(200).nullable().optional()
- `owner_email`: z.string().email().nullable().optional() — use `.or(z.literal(''))` to allow clearing
- `owner_phone`: z.string().max(50).nullable().optional()
- `web_design_tier`: z.enum(['basic', 'advanced']).nullable().optional()
- `domain`: z.string().max(500).nullable().optional()
- `vercel_project_url`: z.string().url().nullable().optional() — use `.or(z.literal(''))` to allow clearing
- `status`: z.enum(['active', 'paused', 'churned']).nullable().optional()
- `monthly_fee`: z.number().min(0).max(99999).nullable().optional()
- `start_date`: z.string().nullable().optional() (ISO date string, validated as DATE)

Export `clientUpdateSchema` and `type ClientUpdateInput = z.infer<typeof clientUpdateSchema>`.

**`lib/actions/client.ts`:**
Add `'use server'` directive at top.
Create `updateClientDetails(businessId: string, data: ClientUpdateInput)` server action:
- Authenticate with createClient(); return `{ error: 'Not authenticated' }` if no user
- Validate businessId is non-empty string
- `clientUpdateSchema.safeParse(data)` — return first fieldError if invalid
- `.update(parsed.data).eq('id', businessId)` on `businesses` table (RLS enforces ownership)
- On error: `console.error` and return `{ error: error.message }`
- On success: `revalidatePath('/clients')` then return `{ success: true }`

**`middleware.ts`:**
Add `"/clients"` to the `APP_ROUTES` array (insert after `"/businesses"`). No other changes to middleware.
