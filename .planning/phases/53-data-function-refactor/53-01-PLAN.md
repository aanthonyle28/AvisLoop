<objective>
Refactor every data function in `lib/data/` to accept an explicit `businessId: string` parameter instead of internally deriving the business from `.eq('user_id', user.id).single()`. This eliminates the PGRST116 crash that occurs when a user has more than one business.

Purpose: This is the foundation of Phase 53. All downstream code (server actions in Plan 53-02, page-level components) depends on these data functions having the correct `(businessId: string, ...)` signature. Once complete, no function in `lib/data/` performs its own user auth or business lookup.

Output: 9 modified files in `lib/data/` with updated function signatures. All callers will break at compile time (intentionally) until Plan 53-02 updates them.
</objective>
