**Part A: Prerequisite — verify user's business and billing state in DB**

1. **Get all businesses for the test user:**
    ```sql
    SELECT b.id, b.name, b.tier, b.stripe_customer_id
    FROM businesses b
    WHERE b.user_id = (SELECT id FROM auth.users WHERE email = 'audit-test@avisloop.com')
    ORDER BY b.created_at;
    ```
    Record: number of businesses, their names, tiers, and stripe status.

2. **Get pooled send count from DB (the ground truth):**
    ```sql
    WITH user_businesses AS (
      SELECT id FROM businesses
      WHERE user_id = (SELECT id FROM auth.users WHERE email = 'audit-test@avisloop.com')
    )
    SELECT COUNT(*) as pooled_send_count
    FROM send_logs
    WHERE business_id IN (SELECT id FROM user_businesses)
      AND is_test = false
      AND created_at >= date_trunc('month', NOW())
      AND status IN ('sent', 'delivered', 'opened');
    ```
    Record the pooled_send_count. This is the number the UI MUST match.

3. **If user has multiple businesses, also get per-business breakdown:**
    ```sql
    WITH user_businesses AS (
      SELECT id, name FROM businesses
      WHERE user_id = (SELECT id FROM auth.users WHERE email = 'audit-test@avisloop.com')
    )
    SELECT ub.name, COUNT(sl.id) as send_count
    FROM user_businesses ub
    LEFT JOIN send_logs sl ON sl.business_id = ub.id
      AND sl.is_test = false
      AND sl.created_at >= date_trunc('month', NOW())
      AND sl.status IN ('sent', 'delivered', 'opened')
    GROUP BY ub.name
    ORDER BY ub.name;
    ```
    Record per-business counts. Sum should equal pooled_send_count from query 2.

4. **Determine expected tier and limit:**
    From query 1, the best (highest priority) tier across all businesses determines the effective tier:
    - trial = 25 sends/month
