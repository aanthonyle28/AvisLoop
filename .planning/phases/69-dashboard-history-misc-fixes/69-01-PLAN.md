**BUG-HIST-01: Fix timezone bug in getSendLogs date range filter.**

In `lib/data/send-logs.ts`, around lines 55-59, the date range filter computes end-of-day using:

```typescript
const endOfDay = new Date(options.dateTo)
endOfDay.setHours(23, 59, 59, 999)
query = query.lte('created_at', endOfDay.toISOString())
```

The problem: `setHours` uses the LOCAL machine timezone. On a UTC-6 server, `setHours(23,59,59,999)` on `2026-03-02` produces `2026-03-03T05:59:59.999Z`, which is correct. But on UTC+5 it would produce `2026-03-02T18:59:59.999Z`, cutting off the last 5 hours of the day.

Replace lines 55-59 with explicit UTC construction:

```typescript
if (options?.dateTo) {
  // Explicit UTC to avoid timezone-dependent setHours()
  const endOfDay = new Date(options.dateTo + 'T23:59:59.999Z')
  query = query.lte('created_at', endOfDay.toISOString())
}
```

This removes the `endOfDay` variable and the `setHours` call. The `dateTo` parameter is always a date string like `"2026-03-02"`, so appending `T23:59:59.999Z` produces a correct UTC timestamp regardless of the server's timezone.

Do NOT change any other `setHours` calls in the file (lines 101, 159 for `startOfMonth` — those use `setDate(1)` + `setHours(0,0,0,0)` for start-of-month which is a different pattern and less sensitive to timezone since it always rounds DOWN).

**BUG-JOBS-01: Add click-to-sort on job table column headers.**

The `job-table.tsx` component already has TanStack's `getSortedRowModel()` and `onSortingChange: setSorting` wired. The problem is that column definitions in `job-columns.tsx` use plain strings for `header` (e.g., `header: 'Customer'`), and the `<th>` rendering in `job-table.tsx` renders them via `flexRender` without any click handler or sort indicator.

**Step 1: Update job-table.tsx header rendering to support sorting.**

Replace the `<th>` rendering in the `<thead>` section (around lines 206-215) with sortable headers. For each header cell, check if the column can be sorted and add click handler + sort indicator if so.

Replace this block:
```tsx
{headerGroup.headers.map(header => (
  <th
    key={header.id}
    className="h-10 px-4 text-left align-middle font-medium text-muted-foreground"
    style={{ width: (header.column.columnDef.meta as Record<string, string>)?.width }}
  >
    {header.isPlaceholder
      ? null
      : flexRender(header.column.columnDef.header, header.getContext())}
  </th>
))}
```

With:
```tsx
{headerGroup.headers.map(header => {
  const canSort = header.column.getCanSort()
  const sorted = header.column.getIsSorted()
  return (
    <th
      key={header.id}
      className="h-10 px-4 text-left align-middle font-medium text-muted-foreground"
      style={{ width: (header.column.columnDef.meta as Record<string, string>)?.width }}
    >
      {header.isPlaceholder ? null : canSort ? (
        <button
          type="button"
          className="flex items-center gap-1 hover:text-foreground transition-colors"
          onClick={header.column.getToggleSortingHandler()}
        >
          {flexRender(header.column.columnDef.header, header.getContext())}
          <span className="inline-flex w-4" aria-hidden="true">
            {sorted === 'asc' ? '↑' : sorted === 'desc' ? '↓' : ''}
          </span>
        </button>
      ) : (
        flexRender(header.column.columnDef.header, header.getContext())
      )}
    </th>
  )
})}
```

This approach:
- Uses `header.column.getCanSort()` to detect sortable columns
- Uses `header.column.getToggleSortingHandler()` for the click handler (TanStack's built-in cycling: unsorted -> asc -> desc -> unsorted)
- Shows `↑` for ascending, `↓` for descending, empty for unsorted
- Wraps sortable headers in a `<button>` for keyboard accessibility
- Non-sortable headers (like "Campaign" which uses `id: 'campaign'` with no `accessorKey`, and "Actions" which has `id: 'actions'`) are rendered without the button
