<task type="auto">
  <name>Task 2: Create billing database migration</name>
  <files>supabase/migrations/00006_add_billing.sql</files>
  <action>
    Create migration file with:

    1. Add stripe_customer_id column to businesses:
       ```sql
       ALTER TABLE public.businesses
         ADD COLUMN stripe_customer_id TEXT UNIQUE;
       ```

    2. Create subscriptions table (schema from RESEARCH.md):
       ```sql
       CREATE TABLE IF NOT EXISTS public.subscriptions (
         id TEXT PRIMARY KEY,                    -- Stripe subscription ID (sub_xxx)
         business_id UUID NOT NULL REFERENCES public.businesses(id) ON DELETE CASCADE,
         status TEXT NOT NULL,                   -- active, past_due, canceled, etc.
         price_id TEXT NOT NULL,                 -- Stripe price ID
         current_period_start TIMESTAMPTZ,
         current_period_end TIMESTAMPTZ,
         cancel_at_period_end BOOLEAN DEFAULT false,
         canceled_at TIMESTAMPTZ,
         created_at TIMESTAMPTZ DEFAULT NOW(),
         updated_at TIMESTAMPTZ DEFAULT NOW()
       );
       ```

    3. Enable RLS and create policy:
       ```sql
       ALTER TABLE public.subscriptions ENABLE ROW LEVEL SECURITY;

       CREATE POLICY "Users view own subscriptions"
       ON public.subscriptions FOR SELECT
       TO authenticated
       USING (business_id IN (
         SELECT id FROM public.businesses WHERE user_id = (SELECT auth.uid())
       ));
       ```

    4. Create index for business lookup:
       ```sql
       CREATE INDEX idx_subscriptions_business_id ON public.subscriptions(business_id);
       ```

    5. Add updated_at trigger (use existing moddatetime pattern):
       ```sql
       CREATE TRIGGER subscriptions_updated_at
         BEFORE UPDATE ON public.subscriptions
         FOR EACH ROW EXECUTE FUNCTION extensions.moddatetime(updated_at);
       ```

    NOTE: Include comment at top: "Run in Supabase SQL Editor"
  </action>
  <verify>SQL file exists with valid syntax (no runtime check - manual apply)</verify>
  <done>Migration file ready for Supabase SQL Editor</done>
</task>
