## Objective

Build the `CreateBusinessWizard` client component — a thin 3-step wizard that reuses existing step UI patterns but calls the scoped server actions from Plan 56-01 instead of the existing `getActiveBusiness()`-based actions. Wire it into the `/onboarding?mode=new` route and add an "Add Business" button on the `/businesses` page that links to it. On wizard completion, call `switchBusiness(newBusinessId)` and redirect to `/dashboard`.

## Critical Safety Rules

1. **NEVER call `saveBusinessBasics()`, `saveServicesOffered()`, `createCampaignFromPreset()`, `acknowledgeSMSConsent()`, or `markOnboardingComplete()`** — these are for first-business onboarding only. Use the Plan 56-01 scoped actions.
2. **Do NOT use localStorage key `'onboarding-draft-v3'`** — that belongs to the existing wizard. Skip draft persistence entirely (the wizard is only 3 steps).
3. **Call `switchBusiness(newBusinessId)` ONLY after the final step succeeds** — not mid-wizard.
4. **The existing `OnboardingWizard` must remain completely unchanged.** This plan adds a NEW component alongside it.

   ```typescript
   interface CreateBusinessWizardProps {
     campaignPresets: CampaignWithTouches[]
   }
   export function CreateBusinessWizard({ campaignPresets }: CreateBusinessWizardProps)
   ```

3. State management:
   - `step` (1 | 2 | 3) — current wizard step
   - `newBusinessId` (string | null) — set after step 1 completes, used by steps 2 and 3
   - `isSubmitting` (boolean) — for final completion

4. **Step 1: Business Setup (basics + services)**
   Build the form inline in this component (or as a sub-component within the same file). The UI should match `BusinessSetupStep` visually:
   - Business name input (required)
   - Phone input (optional)
   - Google review link input (optional)
   - Separator

   - "Other" custom service names input (shows when "other" is selected)
   - Error display
   - "Continue" button

   **On submit:**
   a. Validate name is not empty and at least one service type is selected
   b. Call `createAdditionalBusiness({ name, phone, googleReviewLink })` — this is the INSERT-only action
   c. If success, store `result.businessId` in `newBusinessId` state
   d. Call `saveNewBusinessServices(newBusinessId, { serviceTypes, customServiceNames })` — saves services to the new business
   e. If both succeed, advance to step 2
   f. If either fails, show error, do NOT advance

5. **Step 2: Campaign Preset Selection**
   Build the preset card UI inline. Match `CampaignPresetStep` visually:
   - Use `CAMPAIGN_PRESETS` constant for card metadata (name, description)
   - Match DB presets with constant by name pattern (same logic as CampaignPresetStep lines 31-36)
   - Sort presets to match CAMPAIGN_PRESETS order
   - Radio-style selection with visual ring
   - "Back" and "Continue" buttons

   **On continue:**
   a. Call `createNewBusinessCampaign(newBusinessId!, selectedPresetId)` — creates campaign scoped to new business
   b. If success, advance to step 3
   c. If fail, show toast error

6. **Step 3: SMS Consent**
   Build the TCPA acknowledgment UI inline. Match `SMSConsentStep` visually:
   - Requirements card with bullet list
   - Checkbox with TCPA acknowledgment text
   - "Back" text link
   - "Complete Setup" button

   **On complete:**
   a. Call `completeNewBusinessOnboarding(newBusinessId!)` — sets SMS consent + onboarding_completed_at
   b. If success, call `switchBusiness(newBusinessId!)` — sets the httpOnly cookie
   c. Then call `router.push('/dashboard')` — redirects to dashboard showing the new business
   d. If fail, show toast error

7. Layout wrapper — match the existing wizard layout exactly:
   ```tsx
   <div className="min-h-screen flex flex-col items-center justify-center px-4 pb-20">
     <div className="w-full max-w-lg space-y-8">
       {/* Step content rendered based on `step` state */}
     </div>
     <OnboardingProgress currentStep={step} totalSteps={3} />
   </div>
   ```

8. Import `OnboardingProgress` from `@/components/onboarding/onboarding-progress`.

**Important design note:** The 3 steps are rendered inline within this single component file (or as small sub-functions) rather than importing the existing step components. This is intentional because the existing step components (`BusinessSetupStep`, `CampaignPresetStep`, `SMSConsentStep`) hardcode calls to `saveBusinessBasics()`, `createCampaignFromPreset()`, and `acknowledgeSMSConsent()` which all use `getActiveBusiness()`. Reusing them would require adding optional callback props to all three — more invasive than copying the UI pattern into the new wizard.

The wizard file will be ~300-400 lines, which is acceptable for a self-contained 3-step wizard.
