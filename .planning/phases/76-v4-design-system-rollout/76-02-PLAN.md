# Plan 76-02: Promote V4 Homepage, Archive V3

## Objective

Make the new V4 Framer Motion landing page the homepage at `/`. Move the old V3 web design page content to an archive. Keep the shared marketing layout for pages that still need it (legal pages), but the homepage gets its own standalone layout.

## Context

Currently:
- `/` → V3 web design landing (uses shared marketing layout + v3 components)
- `/new` → V4 Framer Motion landing (standalone layout, self-contained)

After this plan:
- `/` → V4 Framer Motion landing (promoted from /new)
- `/new` → removed entirely

## Tasks

<task id="1">
Move the V4 landing page to become the homepage:

1. Copy the content of `app/new/page.tsx` to `app/(marketing)/page.tsx`, replacing the old V3 homepage entirely
2. The page must NOT use the shared marketing layout's nav/footer — it has its own. To achieve this, the page component should render its own `V4Nav` and `V4Footer` from the shared v4 components (extracted in 76-01), effectively overriding the layout's nav/footer visually.

   **Approach:** Since the `(marketing)` layout wraps with nav + main + footer, and we can't easily opt out per-page, the simplest approach is:
   - Make `app/(marketing)/page.tsx` a `'use client'` page that renders the full V4 page content
   - The marketing layout's nav/footer will still render, but the V4 page's own Nav (fixed, z-50) will overlay the layout nav
   - Add a wrapper div that pushes content to cover the layout footer: `<div className="relative z-10 bg-background">`
   - The V4 Footer renders inside this wrapper

   Alternatively and more cleanly: move the homepage OUT of the `(marketing)` group route entirely — create `app/(home)/page.tsx` with its own `app/(home)/layout.tsx` (minimal, no shared nav/footer). This is the cleaner approach.
</task>

<task id="2">
Create `app/(home)/layout.tsx` — a minimal layout for the homepage only:

```tsx
export default function HomeLayout({ children }: { children: React.ReactNode }) {
  return <>{children}</>;
}
```

No shared nav, no shared footer. The page handles everything.
</task>

**IMPORTANT — Nav links must include these page links (not hash anchors):**

Similarly, the V4Footer must link to both `/reputation` (as "Reviews") and `/client-portal` (it already does in the current `/new` page — verify both are still there).
</task>
