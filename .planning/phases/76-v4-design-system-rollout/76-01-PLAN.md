# Plan 76-01: Extract Shared V4 Design System Components

## Objective

Extract the reusable design system pieces from `app/new/page.tsx` (the 1100-line monolith) into shared component files so every marketing page can use them without duplicating code. This is the foundation ‚Äî all subsequent plans import from these files.

## Context

`app/new/page.tsx` contains Nav, Footer, FloatingShapes, Marquee, AnimatedNumber, and section patterns all inline. These need to become importable components. The current page stays functional throughout ‚Äî we extract, not rewrite.

## Tasks

<task id="1">
Create `components/marketing/v4/nav.tsx` ‚Äî extract the Nav component from `app/new/page.tsx`.

The Nav must be a standalone `'use client'` component that:
- Accepts `links` as a prop: `{ label: string; href: string }[]`
- Accepts an optional `calendlyUrl` prop (defaults to the current CALENDLY constant)
- Keeps the transparent ‚Üí frosted glass scroll behavior
- Keeps the full-screen mobile overlay menu
- Keeps the AvisLoop logomark SVG inline
- Exports as a named export: `export function V4Nav(...)`

Do NOT remove the Nav from `app/new/page.tsx` yet ‚Äî just create the shared component.
</task>

<task id="2">
Create `components/marketing/v4/footer.tsx` ‚Äî extract the Footer component.

The Footer must:
- Accept optional `links` prop for customizing the right-side links (defaults to Privacy, Terms, Reviews, Client Portal)
- Keep the logomark SVG + copyright
- Export as `export function V4Footer(...)`
</task>

<task id="3">
Create `components/marketing/v4/shared.tsx` ‚Äî extract reusable visual building blocks:

1. `FloatingShapes` ‚Äî the animated geometric shapes background (accepts optional `className` for positioning)
2. `AnimatedNumber` ‚Äî the counting number ticker component
3. `MarqueeRow` ‚Äî the infinite scrolling pill row (accepts `items`, `reverse`, `accent` props)
4. `AccentBar` ‚Äî the small animated accent line used as section headers (the `motion.div` with `width: 0 ‚Üí 48` animation)
5. `SectionDivider` ‚Äî the horizontal rule with centered label pattern

All must be `'use client'` components. Export each as named exports.
</task>

<task id="4">
Create `components/marketing/v4/sections.tsx` ‚Äî extract the generic reusable sections:

1. `V4Stats` ‚Äî the dark stats strip with AnimatedNumber, header text, and accent glow (accepts `stats` array and optional `heading`/`subheading`)
2. `V4Testimonials` ‚Äî the single-rotating-quote carousel (accepts `testimonials` array)
3. `V4FAQ` ‚Äî the dark-bg accordion with Plus‚ÜíX rotation (accepts `faqs` array as `[question, answer][]`)

These are sections that multiple pages will reuse with different data.
</task>

<task id="5">
Update `app/new/page.tsx` to import from the new shared components instead of defining them inline. Replace the inline definitions of Nav, Footer, FloatingShapes, AnimatedNumber, MarqueeRow, Stats, Testimonials, and FAQ with imports from the new files. The page should render identically ‚Äî this is a pure refactor.

Verify: `pnpm lint && pnpm typecheck` must pass. The `/new` page must render the same as before.
</task>
