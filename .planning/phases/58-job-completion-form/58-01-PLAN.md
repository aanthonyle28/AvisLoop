**4. Create `lib/actions/form-token.ts`:**
This is a `'use server'` module with two exported async functions:

a) `generateFormToken()`:
   - Call `getActiveBusiness()` -- return `{ error: 'Business not found' }` if null
   - Check if business already has a form_token: query with `createClient()`, select `form_token` where id = business.id
   - If token already exists, return `{ token: existingToken }` (idempotent)
   - Generate token: `randomBytes(24).toString('base64url')` from Node.js `crypto`
   - Update businesses table: `.update({ form_token: token }).eq('id', business.id)`
   - `revalidatePath('/settings')` on success
   - Return `{ token }` or `{ error: 'Failed to generate token' }`

b) `regenerateFormToken()`:
   - Call `getActiveBusiness()` -- return `{ error: 'Business not found' }` if null
   - Generate NEW token: `randomBytes(24).toString('base64url')`
   - Update businesses table (overwrites old token, invalidating old URL)
   - `revalidatePath('/settings')` on success
   - Return `{ token }` or `{ error: 'Failed to regenerate token' }`

Both functions use `createClient()` (auth-scoped) since they're called from the Settings page by an authenticated user.

d) Campaign enrollment (service-role version of `enrollJobInCampaign`):
   - Query active campaign matching service type: `supabase.from('campaigns').select('*, campaign_touches(*)').eq('business_id', businessId).eq('status', 'active').or(\`service_type.eq.${serviceType},service_type.is.null\`).order('service_type', { ascending: false, nullsFirst: false }).limit(1).maybeSingle()`
   - If no campaign found: return success (job created, no enrollment needed)
   - Find touch 1: `campaign.campaign_touches?.find(t => t.touch_number === 1)`
   - If no touch 1: return success (campaign misconfigured, skip enrollment)
   - Get business timing: query `businesses` for `service_type_timing` by businessId
   - Calculate delay: `timing?.[serviceType] || touch1.delay_hours`
   - Calculate scheduled time: `new Date(Date.now() + delayHours * 60 * 60 * 1000)`

   **Conflict detection (service-role version):**
   - Check for active enrollment: `supabase.from('campaign_enrollments').select('id, current_touch, campaigns:campaign_id(name)').eq('customer_id', customerId).eq('business_id', businessId).in('status', ['active', 'frozen']).limit(1).maybeSingle()`
   - If active enrollment exists: set `enrollment_resolution: 'conflict'` and `conflict_detected_at` on the job, skip enrollment, return success (job still created)
   - Check review cooldown: query `businesses` for `review_cooldown_days`, then query `campaign_enrollments` for recent stopped enrollments with `stop_reason` in `['review_clicked', 'feedback_submitted']` within cooldown window

   - On successful enrollment: clear any `enrollment_resolution` on the job

   No `revalidatePath()` calls -- this is a public endpoint with no auth context. Dashboard user sees updates on next page load.

e) Return `{ success: true, jobId: job.id }`

**1. Create `app/api/complete/route.ts`:**
Follow the exact pattern of `app/api/review/rate/route.ts` -- public, rate-limited, service-role API route.

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { createServiceRoleClient } from '@/lib/supabase/service-role'
import { checkPublicRateLimit } from '@/lib/rate-limit'
import { publicJobSchema } from '@/lib/validations/public-job'
import { createPublicJob } from '@/lib/actions/public-job'
```

`export async function POST(req: NextRequest)`:

a) Rate limit by IP:
   ```
   const ip = req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() || 'unknown'
   const rateLimitResult = await checkPublicRateLimit(ip)
   if (!rateLimitResult.success) {
     return NextResponse.json({ error: 'Too many requests' }, { status: 429 })
   }
   ```

b) Parse and validate body:
   ```
   const body = await req.json()
   const parsed = publicJobSchema.safeParse(body)
   if (!parsed.success) {
     return NextResponse.json({ error: 'Validation failed', details: parsed.error.flatten() }, { status: 400 })
   }
   ```

c) Resolve business from token (service-role -- no RLS):
   ```
   const supabase = createServiceRoleClient()
   const { data: business } = await supabase
     .from('businesses')
     .select('id, service_types_enabled')
     .eq('form_token', parsed.data.token)
     .single()

   if (!business) {
     return NextResponse.json({ error: 'Invalid form' }, { status: 404 })
   }
   ```

d) Validate service type against business's enabled types:
   ```
   const enabledTypes = business.service_types_enabled as string[] | null
   if (enabledTypes && enabledTypes.length > 0 && !enabledTypes.includes(parsed.data.serviceType)) {
     return NextResponse.json({ error: 'Service type not available for this business' }, { status: 400 })
   }
   ```

e) Call `createPublicJob()`:
   ```
   const result = await createPublicJob({
     businessId: business.id,
     customerName: parsed.data.customerName,
     customerEmail: parsed.data.customerEmail || undefined,
     customerPhone: parsed.data.customerPhone || undefined,
     serviceType: parsed.data.serviceType,
     notes: parsed.data.notes || undefined,
   })
   ```

f) Return response:
   - If `result.success`: return `NextResponse.json({ success: true }, { status: 201 })`
   - If `!result.success`: return `NextResponse.json({ error: result.error || 'Failed to create job' }, { status: 500 })`

g) Wrap entire handler in try/catch:
   - Catch ZodError: return 400 with details
   - Catch generic error: log error (no PII), return `{ error: 'Internal server error' }` with 500

UI structure (follows existing Settings section patterns -- bordered card with heading):
```tsx
<section className="border border-border rounded-lg p-6 bg-card shadow-sm">
  <h2 className="text-xl font-semibold mb-2">Job Completion Form</h2>
  <p className="text-muted-foreground mb-4">
    Share this link with technicians so they can submit completed jobs on-site — no AvisLoop account needed.
  </p>

  {token ? (
    // Token exists: show URL + copy button + regenerate button
    <div className="space-y-3">
      <div className="flex items-center gap-2">
        <Input
          readOnly
          value={`${window.location.origin}/complete/${token}`}
          className="font-mono text-sm"
        />
        <Button variant="outline" size="icon" onClick={handleCopy} aria-label="Copy form URL">
          {copied ? <Check size={16} /> : <Copy size={16} />}
        </Button>
      </div>
      <div className="flex items-center gap-3">
        <Button variant="ghost" size="sm" onClick={handleRegenerate} disabled={isGenerating}>
          {isGenerating ? <Loader2 className="animate-spin" size={14} /> : <ArrowsClockwise size={14} />}
          <span className="ml-1.5">Regenerate link</span>
        </Button>
        <p className="text-xs text-muted-foreground">
          Regenerating invalidates the current link.
        </p>
      </div>
    </div>
  ) : (
    // No token: show generate button
    <Button onClick={handleGenerate} disabled={isGenerating}>
      {isGenerating ? <Loader2 className="animate-spin" size={16} /> : <LinkSimple size={16} />}
      <span className="ml-2">Generate form link</span>
    </Button>
  )}
</section>
```

- `handleCopy`: `navigator.clipboard.writeText(url)`, set `copied` to true, reset after 2 seconds

Icons: Use Phosphor icons (Copy, Check, ArrowsClockwise, LinkSimple, Loader2 from lucide for spinner consistency).
