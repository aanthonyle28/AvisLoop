<task type="auto">
  <name>Task 1: Create search and submit API routes</name>
  <files>app/api/audit/search/route.ts, app/api/audit/submit/route.ts</files>
  <action>
  Create `app/api/audit/search/route.ts`:
  - POST handler following the pattern in `app/api/review/rate/route.ts`
  - Extract IP from `x-forwarded-for` header (split on comma, trim, fallback to 'unknown')
  - Call `checkAuditRateLimit(ip)` — return 429 with `{ error: 'Daily audit limit reached. Try again tomorrow.', remaining: 0 }` if not success
  - Parse body with Zod schema: `businessName` (string, min 2, max 100, trim), `city` (string, min 2, max 100, trim)
  - Call `searchBusiness(businessName, city)` from `@/lib/audit/places-client`
  - If null result, return 404 with `{ error: 'No business found on Google Maps. Check the name and city.' }`
  - Call `computeReputationScore(result.rating, result.userRatingCount)` from `@/lib/audit/scoring`
  - Return 200 JSON: `{ place: { placeId, displayName, formattedAddress }, grade, score, rating: result.rating, reviewCount: result.userRatingCount }`
  - NOTE: Return rating and reviewCount in the search response (shown in preview). These are NOT persisted — they're live from the API call.
  - Wrap in try/catch. On error, log to console.error and return 500 `{ error: 'Failed to look up business' }`
  - On Zod error, return 400 `{ error: 'Invalid input' }`

  Create `app/api/audit/submit/route.ts`:
  - POST handler
  - Parse body with Zod schema: `email` (string, email), `businessName` (string, min 2, max 100, trim), `city` (string, min 2, max 100, trim)
  - Extract IP from `x-forwarded-for`
  - Call `searchBusiness(businessName, city)` to re-fetch live data from Places API (cannot use cached data per Google TOS)
  - Call `computeReputationScore(result.rating, result.userRatingCount)`
  - Use `createServiceRoleClient()` (NOT anon client — service role can INSERT even without user session, and can read back the ID with `.select('id').single()`)
  - Insert into `audit_reports`: business_name, city, place_id, place_display_name, place_address, score, grade, rating_snapshot, review_count_snapshot, gaps_json (as JSON array), lead_email
  - If insert error, return 500 `{ error: 'Failed to save report' }`
  - Insert into `audit_leads`: email, business_name, city, report_id (from previous insert), ip_address
  - Lead insert is fire-and-forget (don't block on error, but log it)
  - Return 200 JSON: `{ reportId: report.id }`
  - Wrap in try/catch with generic 500 error

  IMPORTANT: Both routes import from `@/lib/audit/places-client` and `@/lib/audit/scoring` — these are server-only modules used in Route Handlers (which are always server-side). This is safe.
  </action>
  <verify>
  Run `pnpm typecheck` — both route files compile without errors.
  Verify search route imports `checkAuditRateLimit` from `@/lib/rate-limit`.
  Verify submit route imports `createServiceRoleClient` from `@/lib/supabase/service-role`.
  Verify both routes use Zod for input validation.
  </verify>
  <done>
  Two API routes created: search (Places API + rate limit + score preview) and submit (email gate + DB insert + reportId return). Both follow existing project patterns for error handling and validation.
